# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 生成前已经视觉检查 `assets/xiaofan/xiaofan-white-redline-head-arrow-sheet-v4.png` 和至少 2-3 张 `assets/examples/*.png`。
- 如果是连续生成多张，每张都重新绑定 v4 角色参考或完整重复角色锁定短块，没有用上一张生成图当身份参考。
- 有小反。
- 小反承担核心动作，不只是装饰。
- 小反是白色未闭合空心椭圆身体、黑色手绘轮廓、体内红斜线、开口处固定黑色箭头。
- 小反的箭头是身体轮廓开口处的末端，不是独立符号、漂浮图标、头饰或道具。
- 小反的箭头只在轮廓开口处，不在脚边、身体下侧、地面、尾巴、工具、标签或背景位置。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面怪诞、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文或英文标注少、短、能读。
- 没有空白气泡、空白说明框、空白按钮或空白标签容器。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 小反像吉祥物、表情包或可爱卡通。
- 小反被画成黑色实心小黑，除非用户明确要求旧版。
- 小反身体被画成完全闭合圆角块，又额外加了一个箭头。
- 小反体内红斜线被画成红色箭头。
- 小反的黑色箭头被画成独立头饰、漂浮图标、手持道具、标签符号或背景装饰。
- 小反的黑色箭头出现在脚边、身体下侧、地面、尾巴、工具或标签位置。
- 连续生成中后几张的小反开始变成另一种角色：闭合白色胶囊、普通白色小人、独立箭头头饰、可爱吉祥物、黑色实心小黑或没有体内红斜线。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 出现空白气泡、空白说明框、空白按钮或空白标签容器。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有荒诞隐喻。
- 和 `composition-patterns.md` 里列出的旧案例构图过于相似。

## 迭代方法

- 太普通：让小反成为动作主体，加入一个奇怪但成立的隐喻。
- 小反不像：先重新查看 v4 动作参考表和 2-3 张 examples，再重写 prompt；不要继续只靠文字加码。
- 连续生成漂移：停止沿用上一张生成图，回到 v4 参考表，重新粘贴完整 `Character identity lock`，单张重生成。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 deadpan、blank serious expression、not cute、not mascot。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像旧案例：保留核心意思，换掉主物件和小反动作。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。
- 空白文字框：直接填短中文或英文，或删掉容器；不要保留待填占位。

## 交付判断

高质量图应该让读者先觉得“有点怪”，然后 1 秒内看懂结构。

如果第一眼像教程页，而不是白纸上的怪诞产品草图，就不合格。
